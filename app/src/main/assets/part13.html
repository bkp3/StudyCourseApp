13. Polymorphism

13.1 Introduction :-

If one task is perform in different ways then it is known as polymorphism.

To achieve polymorphism we use :-

  • Method Overloading
  • Method overriding

13.2 Method Overloading :-

It is also known as Compile time polymorphism or static binding or early binding.

In compile time polymorphism, compiler will distinguish amongst functions with same names but different signatures at compile time itself.

In method overloading the methods name are same but types of argument or no. of arguments are different.

Return type of mehod does not play any role in method overloading.

Example :- 
class Demo {
    void show(int a){
        System.out.println(a);
    }
    void show(int a, int b){
        System.out.println(a + " " + b);
    }
    void show(String str){
        System.out.println(str);
    }
    public static void main(String[] args) {
        Demo obj = new Demo();
        obj.show(5);
        obj.show(5, 10);
        obj.show("Hello");
    }
}

Output:- 
        5
        5 10
        Hello

13.3 Method Overriding :-

It is also known as Run time polymorphism or Dynamic binding or late binding.

In method overriding method of subclass has the same ame and no. of argments or types of arguments are same as a method.

Both signature and return must be the same as super class.

final, static, private can not override.

Example :- 
class Parent {
    void show() {
        System.out.println("Parent show");
    }
    void display() {
        System.out.println("Parent display");
    }
}

class Child extends Parent{
     void show() {
        System.out.println("Child show");
    }
    void display() {
        System.out.println("Child display");
    }
     public static void main(String[] args){
        Parent p =new Parent();
        p.show();
        p.display();
        Child ch = new Child();
        ch.show();
        ch.display();
     }
}

Output:- 
        Parent show
        Parent display
        Child show
        Child display
