Object Oriented Programming Concepts:

1.3 What is Structure Oriented Programming?

In structure oriented/procedure oriented programming language a programmer uses procedure or function to perform a task.

In this approach the main task is divided into several sub task called function and which are controlled by the main function.

Example: C, ALGOL, PASCAL, FORTRAN etc

1.4 What is object oriented programming?

Object Oriented Programming is a design pattern which is based on the concept of class and object.

In Object Oriented Programming the main task is divided in to several modules called classes and each class have its own data and function to perform the task.

Example: C++, Simula, Smalltalk, Java etc.

1.5 Drawbacks of Structure Oriented Programming :-

  • No security for data.
  • No better memory management.
  • Difficult to implement today's client requirements.
  • No structure or code reusability. Hence time of development, testing and length of program increases.
  • As length of application increases it causes slow performances.
  • Code maintenance and enhancements are difficult.
  • No proper way method for Exception handling.

1.6 Difference between Structure Oriented and Object Oriented Programming Language :-

Structure Oriented Object Oriented
Prime focus is on function and procedures that operates on data. It is centered on the concept of class and object.
Large program is divided into smaller program known as function. Main program is divided into smaller program called class and each class contain its own function.
Data move openly around the system from function to funtion. Data is hidden and cannot be accessed by external function.
Functions transform data from one form to another. Objects are communicated with each other through function.
Follows Top-down approach in program design. Follows Bottom-up approach in program design.
Example: C, ALGOL, PASCAL, FORTRAN etc. Example: C++, Simula, Smalltalk, Java etc.

1.7 Features of Object Oriented Programming Systems :-

The following are the features of object oriented programming systems(OOPs).

  • Class and Object
  • Data Abstraction
  • Encapsulation
  • Inheritance
  • Ploymorphism

  1. Object:

    • Objects are the run time entities that exist physically.
    • Objects are the instance of the class
    • The object will have its own states and access to all of the behaviors defined by its class.
    • State of a class means its data member and instance variable which are defined with in a class.
    • Behavior means methods, where class logic is stored. They are where algorithm get executed, and data gets manipulated.

  2. Class:

    • Class is a collection of similar type of object.
    • Class is a model for creating object.
    • Once class is created you can create multiple objects of that class.
    • Example: For example, let us take an object car, which belongs to Vehicles class.Car is an object because it exists physically and which have property like wheels, steering, chassis etc. And function/behavior likes driving, racing. Similrly, bus, bike etc. are the objects of Vehicle class.

  3. Data Abstraction:

    • Hiding of data from the user is known as data abstraction.
    • Data abstraction is a process of representing the essential features without knowing the background details.
    • In this case we can hide unnecessary data and expose the necessary data to the user.
    • Example: For example, the car is an object which has some parts like engine, break and steering etc. The user of the car is the driver should know how to drive the car and does not require any knowledge to know how engine works or how break works.

  4. Encapsulation:

    • Binding of data is known as encapsulation.
    • Wrapping up off data and function in to a single unit is known as encapsulation.
    • Example: For example, if we create a class then it contains data member and member function. Thus a class is an example of encapsulation since it binding the data and behavior together.

  5. Inheritance:

    • Inheritance is the property of a class which acquire all the properties of parent class.
    • Inheritance create a new class from existing class or we may call inheritance is the existing properties of class. A good example of inheritance is parent's child relationship.

  6. Polymorphism:

    • Polymorphism is one of the primary characteristics of object oriented programming.
    • Poly means many ad morph means form. Thus, polymorphism refers to being able to use many forms of a type without regard to the deatils.

1.8 JVM :-

  • JVM stands for java virtual machine.
  • JVM is not a machine or a hardware device it is purely a software application.
  • JVM is the heart of entire Java program execution process.
  • JVM plays the main role for making the java program portable.
  • JVM is responsible for taking the .class file and converting each byte code instruction in to the machine language instruction tha can be executed by the microprocessor.

    • Execution Process:-

  • First of all, Java Program in Compiles by the JAVAC Compiler and which generate Class file.
  • The Class file contents byte code instruction and which are the optimized instruction of Java code.
  • The byte code instuction can only understand by the JVM but not by the programmer nor by the Machine.
  • Inside the JVM JAVA interpreter will convert byte code instruction to machine code and can be understand by the processor.

    • How Does the JVM work?

  • As shown in the architecture diagram. the JVM is divided into three main subsystems.
    1. Class Loader Subsystems
    2. Runtime Data Area
    3. Execution Engine

  1. Class Loader Subsystem :-

    2. Java's dysnamic class loading functionality is handled by the class loader subsystem. It loads, links and initilizes the class file when it refers to a class for the first time at runtime, not compile time.

    1. Loading :-

      2. Classes will be loaded by this component, Boot Strap class Loader, Extension class loader and Application class loader are the three clas loader which will help in achieving it.

    2. Linking :-

      3. Verify :- Bytecode verifier will verify whether the generated bytecode is proper or not if verification fails we will get the verification error

      Prepare :- For all static variables memory will be allocated and assigned and assigned with default values.

      Resolve :- All symbolic memory references are replaced with the original references from method area.

    3. Initialization :-

      4. This is the final phase of class loading, here all static variables will be assigned with the original values and the static block wil be executed.

  2. Runtime Data Area :-

    3. The runtime data area is divided into 5 major components :-

    1. Method Area :-

      All the class level data will be stored here, including static variables. There is only one method area per JVM and it is a shared resource.

    2. Heap Area :-

      All the objects and their corresponding instance variables and arrays will be stored here. There is also one Heap Area per JVM. Since the method and heap area share memory for multiple threads, the data stored is not thread safe.

    3. Stack Area :-

      For every thread, a seperate runtime stack will be created. For every method call, one entry will be made in the stack memory. The stack memory which is called as Stack Frame. All local variables will be created in the stack memory. The stack area is thread safe since it is not a shared resource.

    4. PC Registers :-

      Each thread will have seperate PC Registers, to hold the address of current executing instruction once the instruction is excuted the PC registers will be updated with the next instructions.

    5. Native Method Stacks :-

      Native Method Stack holds native method information. For every thread, a seperate native method stack will be created.

  3. Execution Engine

    4. The bytecode which is assigned to the Runtime Data Area will be executed by the Execution Engine. The Execution Engine reads the bytecode and executes it piece by piece.

    1. Interpreter :-

      The interpreter interprets the bytecode faster, but executes slowly. The disadvantage of the interpreter is that when one method is called multiple times, every time a new interpretation is required.

    2. JIT Compiler :-

      The JIT Compiler neutralizes the disadvantages of the interperter. The Execution Engine will be using the help of the interperter in coonverting byte code, but when it finds repeated code it uses the JIT compiler, which compiles the entire bytecode and changes it to native code. This native code will be used directly for repeated method calls, which improve the performance of the system.

    3. Garbage Collector :-

      Collects and removes unreferenced objects. garbage Collection can be triggered by calling "System.gc()", but the execution is not guaranteed. Garbage collection of the JVM collects the objects that are created.

1.11 Meaning of public static void main(String args[]) :-

  • main() :- main() method is the entry point of java program execution and from the main() method the program execution will start.
  • When you run any java program, first of all the JVM will search for the main() method and the main() method should be always should be always in the following format- public static void main(String args[]){}.
  • public :- public is an access specifier and if a variable or method is public then it will be accessible to everybody. if Java's main() method is public then it will be available to JVM, since JVM is present outside of your java program.
  • Static :- If java's variable or method is declared as static then it will accessible to other before creating the instance of the class. Java's main() method is always static since, it should be available to JVM creating the instance (object) of the class.
  • void :- java's main() method doesn't return any value.
  • main() :- main() is the name of the method.
  • Java's main() method's argument is always string type means, java program can access only string type of variable passes on command line.
