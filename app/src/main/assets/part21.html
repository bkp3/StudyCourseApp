21. Collection

21.1 Introduction :-

Collection framework is basically implementation of data structure. If we want to do operations on data structure objects like array, linkedlist, stack. then we have to write logic for this. By using collection framework we can do it easily beacuase of predefined methods.

Collection framework has :-

  1. Interface and its implementation i.e. classes
  2. Algorithm

List of interfaces :-

  1. ArrayList
  2. LinkedList
  3. Vector
  4. Stack
  5. PriorityQueue
  6. ArrayDequeue
  7. HashSet
  8. LinkedHashSet
  9. TreeSet
  10. HashMap
  11. LinkedHashMap
  12. TreeMap
  13. HashTable

21.2 ArrayList :-

Dynamic Array

Duplicate elements

Maintain insertion order

Non-Synchronized

Elements Randomly accessed

Example :- 
import java.util.*;
class Demo {
    public static void main(String[] args) {
        ArrayList< String > arr = new ArrayList< String >();
        arr.add("Ravi");
        arr.add("Vijay");
        arr.add("Ravi");
        arr.add("Ajay");
        for(int i = 0 ; i < arr.size() ; i++){
            System.out.println(arr.get(i));
        }
    }
}

Output:- 
        Ravi
        Vijay
        Ravi
        Ajay

No duplicate elements.

21.11 HashMap :-

It contains values based on the key.

It contains only unique keys.

It may have one null key and mulitpl null values.

No Maintain insertion order.

Non-Synchronized.

Maintains order at key level(Ascending order).

Example :- 
import java.util.*;
class Demo {
    public static void main(String[] args) {
        HashMap< Integer, String > arr = new HashMap();
        arr.put(101, "one");
        arr.put(103, "two");
        arr.put(105, "three");
        arr.put(104, "four");
        arr.put(102, "five");
        for( map.Entry x: arr.entrySet()){
            System.out.println(x.getKey() + " " + x.getValue());
        }
    }
}

Output:- 
        101 one
        102 five
        103 two
        104 four
        105 three
