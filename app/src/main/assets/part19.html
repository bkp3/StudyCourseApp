19. Multithreading

19.1 Introduction :-

Multithreading in java is a process of executing multiple threads simultaneously.

Thread is basically a light weight sub-process, a smallest unit of processing.

Multiprocessing and multithreading both are used to achives multitasking.

But we use multithreading than multiprocessing beacause threads share a common memory area. They don't allocate separate memory are so saves memory takes and context-switching between the threads takes less time than process.

Java multithreading is mostly used in games, animation etc.

19.2 Advantages of Multithreading :-

It doesn't block the user because threads are independant and you can perform multiple operation at same time.

You can perform many operation together so it saves time.

Threads are independant so it does not affects other threads if exception occurs in a single thread.

19.3 Multitasking :-

Multitasking is a process of executing multiple tasks simultaneously.

We use multitasking to utilize the cpu.

Multitasking can be achieved by two ways. :-

  1. Process-based multitasking (Multiprocessing)
  2. Thread-based multitasking (Multithreading)

19.4 Process-based multitasking :-

Each process has its own address in memory. ie each process allocates separate memory area.

Process is heavy weight.

Cost of communication is high.

19.5 Thread-based multitasking :-

Threads share the same address space.

Thread is light weight.

Cost of communication btween the thread is low.

19.6 Thread :-

Thread represents a separate path of execution of group of statement.

In java program, If we write group of statement then the statement is executed by JVM one by one. This execution is called a thread because JVM uses a thread to execute three statement.

Thread is flow of execution.

We can define a thread :-

  1. by extending thread class.
  2. by implementing runnable interface.

i. By extending thread class.

Example :- 
class Demo extends Thread{
    void run(){
        for(int i = 0; i < 5 ; i++){
            System.out.println("Hello World");
        }
    }
    public static void main(String[] args) {
        Demo obj = new Demo();
        Thread t = new Thread(obj);
        t.start();
    }
}

Output:- 
        Hello World
        Hello World
        Hello World
        Hello World
        Hello World

ii. By implementing runnable interface.

Example :- 
class Demo implements Runnable{
    void run(){
        for(int i = 0; i < 5 ; i++){
            System.out.println("Hello World");
        }
    }
    public static void main(String[] args) {
        Demo obj = new Demo();
        Thread t = new Thread(obj);
        t.start();
    }
}

Output:- 
        Hello World
        Hello World
        Hello World
        Hello World
        Hello World

19.7 Life cycle of a thread (Thread state) :-

There are five states in thread life cycle in java.

The life cycle of the thread in java is controlled by JVM.

Java thread states are as follows.

  1. New
  2. Runnable
  3. Running
  4. Non-Runnable(waiting)
  5. Terminated

i. New :- The thread is in new state if we create an object of thread class but before the calling of start() method.

ii. Runnable :- The thread is in runnable state after calling of start() method.

iii. Running :- The thread is in running state if the thread schedular has selected it.

iv. Waiting :- This is the state when the thread is still alive but it is currently not eligible to run.

v. Terminated :- A thread is in terminated or dead state when its run() method exists.
