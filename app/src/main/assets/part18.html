18. Exception Handling

18.1 Exception :-

Exception is an abnormal condition that occurs during the execution of a program & due to which normal flow of the program's insrtuction may distrupt.

Example :- 
class Demo {
    public static void main(String[] args) {
        System.out.println("Hello");
        System.out.println(5/0);
        System.out.println("Hello Again");
    }
}

Output:- 
        Hello
        Error :- Exception in main thread java.lang.ArithmeticException : / by zero

18.2 Exception Handling :-

It is a mechanism to handle the exception.

Exception Handling is to maintain the normal flow of the application and to prevent abnormal termination of program.

Some common exception

  • Arithmetic Exception :-
    int a = 5/0;
  • Null Pointer Exception :-
    String str = null;
    System.out.println(s.length());
  • ArrayIndexOutOfBounds Exception :-
    int []arr = new int[5];
    System.out.println(arr[6]);
  • Number Format Exception :-
    String str = "12x";
    int i = Integer.parseInt(str);

Exception Handling is managed via five keywords :-

  1. try
  2. catch
  3. finally
  4. throw
  5. throws

18.3 try :-

try block contains multiple statements that may cause different types of exception.

18.4 catch :-

Catch blcok used to catch the exception throw by the try block.

Example :- 
class Demo {
    public static void main(String[] args) {
        System.out.println("Hello");
        try{
            System.out.println(5/0);
        }
        catch(Exception e){
            System.out.println(e);
        }
        System.out.println("Hello Again");
    }
}

Output:- 
        Hello
        java.lang.ArithmeticException: / by zero
        Hello Again

18.5 finally :-

finally is always excuted whether exception is handled or not.

Java finally block follows try or catch block.

Example :- 
class Demo {
    public static void main(String[] args) {
        try{
            int a = 5;
            int b = 0;
            System.out.println(a/b);
        }
        catch(Exception e){
            System.out.println(e.getMessage());
        }
        finally{
            System.out.println("finally block");
        }
        System.out.println("Program end");
    }
}

Output:- 
        / by zero
        finally block
        Program end

18.6 throw :-

Java throw keyword is used to explicitly throw an exception.

throw is used within the method.

we can not throw multiple exceptions.

Example :- 
class Demo {
    void fun(int age){
        if(age < 18){
            throw new ArithmeticException("not eligible for voting");
        } else{
            System.out.println("Eligible for voting");
        }
    }
    public static void main(String[] args) {
        Demo obj = new Demo();
        obj.fun(15);
    }
}

Output:- 
        Exception in thread main java.lang.ArithmeticException not eligible for voting

18.7 throws :-

Java throws keyword is used to declare an exception.

throws is used with the method signature.

We can declare multiple exception.

Example :- 
class Demo {
    int fun(int a, int b) throws ArithmeticException {
        int c = a/b;
        return c;
    }
    public static void main(String[] args) {
        Demo obj = new Demo();
        try{
            System.out.println(obj.fun(15,0));
        }
        catch(ArithmeticException e){
            System.out.println("You should not divide a no. by zero.");
        }
    }
}

Output:- 
        You should not divide a no. by zero.
