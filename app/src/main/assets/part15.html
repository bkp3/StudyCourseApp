15. Abstraction

15.1 Introduction :-

Hiding the internal implementation and only showing its functionality is known as abstraction.

To achieve abstaction, we use

  • Abstract class
  • Interface

15.1 Abstract class :-

Abstract method => method without body.

Non-Abstract method => method with body.

A class that is marked with abstract keyword is known as abstract class.

Abstrac class can have abstract and non-abstract method.

static method & constructor can not be declare as abstract.

Abstract class and abstract method can not declare as final.

Example :- 
abstract class Animal{
    abstract void talk();
}

class Dog extends Animal{
    void talk(){
        System.out.println("Bark");
    }
}

class Cat extends Animal{
    void talk(){
        System.out.println("Mew");
    }
}

class Demo {
    public static void main(String[] args) {
        Animal obj1 = new Dog();
        Animal obj2 = new Cat();
        obj1.talk();
        obj2.talk();
    }
}

Output:- 
        Bark
        Mew

15.3 Interface :-

Interface can have only abstract methods.

All Methods are by default public.

only public, static and final variables.

No constructor

Method can not be private, protected, final, synchronized.

Example :- 
interface Animal{
    public void talk();
}

class Dog implemtents Animal{
    void talk(){
        System.out.println("Bark");
    }
}

class Cat implements Animal{
    void talk(){
        System.out.println("Mew");
    }
}

class Demo {
    public static void main(String[] args) {
        Animal obj1 = new Dog();
        Animal obj2 = new Cat();
        obj1.talk();
        obj2.talk();
    }
}

Output:- 
        Bark
        Mew
